Mac设置环境变量(adb+NDK+sdk)

发表于 2015-05-15   |   分类于 Mac系统   |  

无论是在windows还是mac系统下，对于程序猿而言，在terminal中输入命令进行一些常用操作是比用鼠标更为快捷的方式。那么为了能够打开terminal就可以直接输入并执行命令，就需要进行环境变量的设置，这样就不必进入命令文件所在目录下执行该命令了。windows下配置环境变量较为简单，在此讲一下mac下配置环境变量的方法和注意事项。

1. 打开terminal，输入cd ~, 进入当前用户home目录

2. 输入touch .bash_profile, 创建bash_profile文件(如果该文件已经存在则可以直接进入第3步)

3. 打开记事本编辑bash_profile文件，示例如下

方式一：用SDK_HOME指代一个通用路径，一处设置，多处使用，结构明晰，然后设置到PATH

export SDK_HOME=“/usr/sdk”             
export PATH=${PATH}:${SDK_HOME}:${SDK_HOME}/tools:/usr/ndk


方式二：直接将各个命令的路径用:连接起来设置到PATH 形如--- export PATH=${PATH}:路径1:路径2:路径3

export PATH=${PATH}:/usr/sdk:/usr/sdk/tools:/usr/ndk

4. 保存修改，输入更新命令source .bash_profile,PATH变量配置完成

如何阅读一本书

发表于 2015-05-12   |   分类于 读书札记   |  

书籍使人进步！阅读使人明智！如何阅读一本书也是一门学问，读懂不同类型的书也是一件不容易的事情。美国教育家查尔斯-范多伦还特地写了一本《如何阅读一本书》来讲述读书的“方法论”，此文为部分书摘，闲来无事翻书，翻到哪儿写到哪儿，未完待续……

阅读一本书的两种层次

你是为了获得资讯而读还是说为了提高自己的理解力而读；后者其实是一种不同理解水平的双方之间的对话，如何缩小理解上的差距是在阅读过程中很重要的一件事，即如何从粗浅的了解推进到深入理解的层次，如何把一个模糊的概念具体化成清晰的模型！

读书是一种自我发现型的学习

读书是一种自我发现型的学习，不需要第三方（如老师）的辅助；在读一本较为高深的书时，建议一路读下去不做停顿，读你能领会的一部分即可，难理解之处可以留到下次读的时候在注意领会，包括查找资料，请教他人等等。检索阅读可以了解书的大致构架（注意书目、说明、注释），略读的目的是了解是否需要进一步读下去！

如何做一个自我要求的读者？

保持清醒，在阅读过程中，不断提出问题，并通过自己的阅读来加以解决这些问题；那么如何来保持清醒呢？那就是在书上做笔记，写下来你的感想或者作者观点的精彩部分，既可以开动自己的大脑，记录自己的思想，也可以帮助你记录作者的思想，表达你与作者之间相同或者相异的观点；主动的阅读本身便是一种思考，而思考倾向于用语言或文字表达出来。由许多规则中才可以养成一个习惯，清楚了解规则并加以高强度练习，才可以真正意义上地掌握一种技能，因为对初学者而言：任何一个复杂的活动，都必须从单一的方面入手，先熟悉基本步骤，最后加以组合，忘却步骤，和谐地进行某一活动，也就是说到后期就不能再拘泥于单一动作，而是一种习惯和感觉的驱使！

一本书的分类

首先你的有一个标准，那么你如何来寻找这样一个标准呢？如果你想简单明白地运用一个规则，那么你必须先使这个规则更加简单明白一些！书有理论性和实用性、小说类和论说类之分，那么区别在哪里呢？那就是：我们不应该只是满足于只知道这是怎么回事，而应该明白这个东西应该怎么运用？再举例，如：你质疑某件事的合理性与有效性，这是理论；而你质疑任何事的目的都可以称之为实用。如何判断一本理论书的具体类别呢？它到底是科学or哲学？这就的看它所强调的是否是超现实的！哲学家利用人类共通经验来思考，以便于向他人论证以及灌输自己的观点；而科学家则是利用个人特殊的经验，研究调查得出观点。他们的不同之处在于依托的经验&&思考方式&&论证问题的方式

透视一本书

从架构的寻找到间短归纳主要内容，达到你可以清晰地向他人讲述一本书在讲些什么；再到你可以按照一种顺序来把握意思一本书的重要部分、列出纲要，串联起来；最后一步就是如何提出作者以及你自己的问题，并加以解决！
